## 🛠️ Tech Stack

### Core Backend & Framework
*   **Language:** Go (Golang) - Leverages Goroutines & Channels for high-performance concurrent processing.
*   **Web Framework:** Fiber v3 - Built on top of `fasthttp`, offering the fastest execution speed and minimal memory footprint in the Go ecosystem.

### Database & Data Layer
*   **Primary Database:** PostgreSQL - Provides full ACID transaction support to guarantee data consistency.
*   **Database Driver:** `pgx/v5` - The fastest PostgreSQL driver for Go, featuring advanced connection pooling optimized for concurrency.
*   **Database Client (SQL-First):** `sqlc` - Compiles raw SQL queries into type-safe Go code, delivering zero runtime overhead performance.
*   **Database Migration:** `golang-migrate` - Manages database schema version control using plain `.sql` files.

### Messaging & Communication
*   **Message Queue:** Apache Kafka - A distributed event streaming platform used to handle high-throughput, asynchronous order processing to prevent system crashes during spikes.
*   **Internal RPC:** gRPC / Protocol Buffers - Facilitates ultra-fast, low-latency synchronous communication between internal microservices.

### DevOps & Infrastructure
*   **Containerization:** Docker & Docker Compose - Spins up the entire infrastructure environment (PostgreSQL, Kafka, Services) with a single command.

## ⚡ Key Highlights for Portfolio

1. **Zero-Overhead DB Layer:** Utilizes `sqlc` coupled with `pgx/v5` to eliminate standard ORM overhead bottlenecks, squeezing out 100% of PostgreSQL's raw capabilities.
2. **Race Condition Prevention:** Completely eliminates double-booking issues during extreme traffic spikes by implementing strict `Pessimistic Locking (SELECT FOR UPDATE)` inside atomic ACID transaction scopes.
3. **High-Throughput Worker Pools:** Implements a decoupled background worker pattern. A customized pool of **Goroutines** consumes events asynchronously from **Apache Kafka** at a throttled rate, protecting the core database from overloading.
